① Conda 环境管理
📦 虚拟环境概念

虚拟环境为每个项目创建独立的 Python 运行空间，避免不同项目的包版本冲突。

类比说明
📦 Conda 环境每个 LLM 项目一个独立"房间"，互不干扰
🎒 不用虚拟环境所有项目共用全局 Python，依赖打架
🛠 核心命令
步骤命令作用
①conda --version确认 Conda 已安装
②conda create -n llm python=3.11 -y创建名为 llm 的环境，Python 3.11
③conda activate llm激活环境（终端出现 (llm) 前缀）
④python --version验证环境中的 Python 版本
⑤pip install 包名在 llm 环境中安装包
💡 关键理解：conda activate llm 之后，pip install 装的包只在这个环境里，不影响其他项目。

📌 字典定义 dict
# 定义字典：键 = 城市名，值 = 温度
city_temp = {"北京": 25, "上海": 20, "广州": 28,
             "深圳": 22, "成都": 26}

dict ≈ Swift 的 [String: Int]，键值对存储

📌 列表推导式 list comprehension
hot_cities = [(city, temp) for city, temp in city_temp.items() if temp > 20]

拆解：

• city_temp.items() → 遍历出 ("北京", 25) 这样的键值对

• for city, temp in ... → 解包赋值

• if temp > 20 → 筛选条件

• (city, temp) → 组装成元组放入新列表

结果：[('北京',25), ('广州',28), ('深圳',22), ('成都',26)]

⚠️ 易错 如果只写 city 而不是 (city, temp)，后面无法取到温度

📌 f-string 格式化 f-string
for city, temp in hot_cities:
    print(f"      {city} 温度为 {temp}℃")

f"..." 中的 {变量} 会自动替换为变量的值

≈ Swift 的 "\(city)"

📌 sorted + Lambda 排序 sorted lambda
hot_cities_sorted = sorted(hot_cities,
    key=lambda x: x[1],  # 按温度排序（x[1] = 温度）
    reverse=True         # 降序（从高到低）
)

拆解：

• sorted(列表, key=函数, reverse=布尔) → 返回新排序列表

• lambda x: x[1] → 匿名函数，对每个元素返回 x[1] 作为排序依据

• reverse=True → 降序（默认升序）

结果：[('广州',28), ('成都',26), ('北京',25), ('深圳',22)]

📌 同步调用 vs 流式调用 关键区别
对比同步 (默认)流式 (stream=True)
工作方式等全部生成完 → 一次性返回一边生成一边返回
代码response = client...create()
print(response.choices[0].message.content)		stream = client...create(stream=True)
for chunk in stream: print(chunk...)
体验等待 → 突然看到全部逐字显示（打字机效果）
适合场景后台处理、批量分析聊天对话、实时交互
流式关键参数：stream=True + flush=True（强制刷新输出）
