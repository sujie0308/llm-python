② 事件循环 (Event Loop)
📌 Python 异步的核心引擎

事件循环 是一个无限循环，不断检查有哪些协程可以继续执行。它运行在单线程上，靠"调度"而不是多线程实现并发。

# 事件循环的简化理解
tasks = [task1, task2, task3]  # 要执行的协程列表

while True:                    # 事件循环
    for task in tasks:
        if task.is_ready():       # 可以继续了吗？
            task.run_one_step()    # 执行一步，直到遇到下一个 await
        # 如果还没准备好（还在等 IO），跳过它，检查下一个
特性事件循环多线程
运行位置单线程多线程
切换成本极低（用户态，~0.01μs）高（内核态，~1μs）
内存开销每个协程 ~几 KB每个线程 ~8MB
适合场景IO 密集（网络、数据库）CPU 密集（计算、渲染）
💡 类比：事件循环就像餐厅的一个服务员（单线程），可以为多桌客人服务——客人点菜后（发请求），服务员去服务其他桌（切换任务），菜好了再回来（await 完成）。
💡 Swift 对比：EventLoop 和 Actor

Python 的事件循环 ≈ Swift Concurrency 中的 Executor（全局调度器）。

区别：Python 的 asyncio 需要手动管理事件循环（asyncio.run()），Swift 的 async/await 是语言内置的，编译器自动处理调度。

📖 任务 3：装饰器原理
① 装饰器本质 — 高阶函数 + 闭包
📌 装饰器 = 接收函数，返回新函数
# 装饰器的本质
def decorator(func):       # ① 接收一个函数
    def wrapper(*args, **kwargs):  # ② 内部定义一个包装函数（闭包）
        # 在调用原函数前/后做额外的事
        result = func(*args, **kwargs)
        return result
    return wrapper             # ③ 返回包装函数

# @ 语法糖等价于：my_func = decorator(my_func)
@decorator
def my_func():
    pass
概念关系
闭包内部函数引用外部变量——一种编程机制
装饰器用闭包包装另一个函数的模式——闭包的典型应用
💡 Swift 对比：装饰器 = 高阶函数

Python 装饰器 ≈ Swift 中接收函数、返回函数的高阶函数：

// Swift 等效
func timer<T>(_ f: @escaping () -> T) -> () -> T {
    return {
        let start = CFAbsoluteTimeGetCurrent()
        let result = f()
        let elapsed = CFAbsoluteTimeGetCurrent() - start
        print("耗时: \(elapsed)s")
        return result
    }
}

// 使用
let slowFunc = timer { slowFunc() }

区别：Python 有 @ 语法糖，Swift 没有直接对应的装饰器语法，但可以用高阶函数实现相同的效果。

② 手写 @timer 装饰器（同步版 + 异步版）
📌 计算函数执行时间
# 同步版 @timer
import time

def timer(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        elapsed = time.time() - start
        print(f"{func.__name__} 耗时: {elapsed:.4f}s")
        return result
    return wrapper

@timer
def slow_func():
    time.sleep(1)
    return "完成"

slow_func()  # 输出：slow_func 耗时: 1.0032s
# 异步版 @async_timer（用在 AsyncLLMClient 中）
import functools

def async_timer(func):
    @functools.wraps(func)
    async def wrapper(*args, **kwargs):
        start = time.time()
        result = await func(*args, **kwargs)  # await 原异步函数
        elapsed = time.time() - start
        print(f"⏱ {func.__name__} 耗时 {elapsed:.2f}s")
        return result
    return wrapper

wrapper 类型def wrapper(...)async def wrapper(...)
调用原函数func(*args)await func(*args)
适用普通函数async def 协程函数
③ @functools.wraps — 保留原函数信息
📌 让装饰后的函数保留原名字和文档
# ❌ 不加 @wraps 的后果
def timer(func):
    def wrapper(*args, **kwargs):
        """我是 wrapper"""
        return func(*args, **kwargs)
    return wrapper

@timer
def hello():
    """我是 hello"""
    pass

print(hello.__name__)  # → "wrapper"  ❌ 不是 "hello"
print(hello.__doc__)   # → "我是 wrapper" ❌ 文档也丢了
# ✅ 加 @wraps 后
import functools

def timer(func):
    @functools.wraps(func)    # ← 复制 func 的元信息到 wrapper
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

print(hello.__name__)  # → "hello" ✅
print(hello.__doc__)   # → "我是 hello" ✅
⚠️ 必须加！写装饰器时永远加上 @functools.wraps(func)，否则调试时所有被装饰的函数都显示为 wrapper，无法区分。
④ 带参数的装饰器 — 三层嵌套
📌 比如 @retry(times=3)
def retry(times: int = 3, delay: float = 0):
    """失败重试装饰器"""
    def decorator(func):           # 中层：接收函数
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for i in range(times):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    if i == times - 1: raise
                    print(f"重试 {i+1}/{times}")
                    if delay: time.sleep(delay)
        return wrapper
    return decorator               # 返回中层

@retry(times=3, delay=1)
def unstable_api():
    ...
💡 三层嵌套的执行顺序：
@retry(times=3) → 先调用 retry(3) 返回 decorator
→ @decorator 再把下面的函数传进去 → 返回 wrapper
最终：unstable_api = wrapper

class DataProcessor:
    def __init__(self, data):
        self.data = data

    @cached_property
    def processed(self):
        print("处理中...")   # 只打印一次
        return [x * 2 for x in self.data]

dp = DataProcessor([1, 2, 3])
print(dp.processed)  # → 计算并返回
print(dp.processed)  # → 直接返回缓存 ✅

# 实例：HTML 标签装饰器
def bold(func):
    @functools.wraps(func)
    def wrapper():
        return f"<b>{func()}</b>"
    return wrapper

def italic(func):
    @functools.wraps(func)
    def wrapper():
        return f"<i>{func()}</i>"
    return wrapper

@bold       # 第二步装饰
@italic     # 第一步装饰
def hello():
    return "Hello"

print(hello())  # → <b><i>Hello</i></b>
# 执行顺序：bold 先 → italic 后 → hello → italic 结束 → bold 结束
