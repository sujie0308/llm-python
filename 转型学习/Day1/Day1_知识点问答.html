1 为什么 AI 开发需要虚拟环境？Conda 和 pip 的区别是什么？ ▼

conda环境

虚拟环境的必要性：不同 AI 项目依赖的 PyTorch/CUDA 版本可能冲突。比如项目 A 要 PyTorch 2.0 + CUDA 11.8，项目 B 要 PyTorch 2.5 + CUDA 12.4。没有虚拟环境就会打架。

Conda vs pip：

 Condapip
管理范围Python + 系统库（如 CUDA）仅 Python 包
环境隔离✅ 原生支持❌ 需要 venv/poetry
依赖解析强（SAT 求解器）弱（可能有冲突）
适用场景数据科学 / ML（PyTorch, GPU）通用 Python
常用命令conda create / installpip install

最佳实践：用 Conda 管理环境 + Python 版本，用 pip 装包（因为 PyPI 上的包更新更快）。即：conda create -n llm python=3.11 -y，然后 pip install torch ...

2 怎么创建一个叫 llm 的 Conda 环境？怎么激活和删除？ ▼

conda环境

创建环境：

conda create -n llm python=3.11 -y

激活环境：

conda activate llm

激活后终端前面会出现 (llm) 标记，所有 python 和 pip 命令都在这个环境里。

查看已有环境：

conda env list

退出环境：

conda deactivate

删除环境：

conda remove -n llm --all

在 Trae 中选择环境：Ctrl+Shift+P → "Python: Select Interpreter" → 选 llm 环境。右下角会显示 Python 3.11.x ('llm')。

3 .env 文件是干什么的？为什么不能上传到 GitHub？ ▼

conda环境

.env 文件用来存敏感配置，比如 API Key、API 地址、数据库密码等。

# .env 文件内容示例
OPENAI_API_KEY=sk-your-key-here
OPENAI_API_BASE=https://api.example.com/v1
MODEL=qwen2.5-7b

Python 用 python-dotenv 加载：

from dotenv import load_dotenv
import os
load_dotenv()                     # 加载 .env 文件
api_key = os.getenv("API_KEY")    # 读取环境变量

为什么不能上传到 GitHub？因为 API Key 是敏感信息，如果上传到公开仓库，别人可以盗用你的额度，造成经济损失。

正确做法：

  • 把 .env 加到 .gitignore 里
  • 创建一个 .env.example 作为模板上传（只写 key 名，不写真实值）

5 Python 的缩进规则是什么？为什么缩进错了程序会崩溃？ ▼

Python语法

Python 用缩进代替了大括号 {} 来表示代码块。

# Swift
if x > 0 {
    print("正数")
} else {
    print("非正数")
}

# Python
if x > 0:
    print("正数")    # 注意缩进 4 个空格
else:
    print("非正数")  # else 和 if 对齐

为什么缩进错了会崩溃？因为缩进在 Python 里不是风格问题，是语法的一部分。缩进不对 = 语法错误。

if x > 0:
print("正数")  # ❌ IndentationError: expected an indented block

规则：

  • 同一代码块的缩进必须一致（通常 4 个空格）
  • 不要混用 Tab 和空格（Python 3 禁止混用）
  • 大多数 IDE 会自动处理缩进，按 Enter 自动加 4 空格
6 Python 的列表（list）和字典（dict）怎么用？和 Swift 的 Array/Dictionary 有什么异同？ ▼

Python语法

列表 list（≈ Swift Array）：

# Python
nums = [1, 2, 3, 4, 5]
nums.append(6)       # 末尾加
nums[0]              # 访问 → 1
nums[-1]             # 倒数第一个 → 6
nums[1:3]            # 切片 → [2, 3]
len(nums)            # 长度 → 6

# 列表推导式（Swift 没有直接等价）
squares = [x**2 for x in range(10) if x % 2 == 0]
# → [0, 4, 16, 36, 64]

字典 dict（≈ Swift Dictionary）：

# Python
scores = {"小明": 95, "小红": 88, "小刚": 72}
scores["小明"]        # 访问 → 95
scores["小华"] = 90   # 新增/修改
"小明" in scores      # 判断 key 是否存在 → True
scores.get("小华", 0) # 安全访问，不存在返回默认值 0
scores.items()        # 遍历 (key, value) 对

# 遍历
for name, score in scores.items():
    print(f"{name}: {score}")

区别：Swift 字典用 ["key": value]，Python 用 {"key": value}（冒号代替等号）。

7 range() 是什么？怎么用它写 for 循环？ ▼

Python语法

range() 生成一个整数序列，通常用在 for 循环中。

# range(stop)         → 0 到 stop-1
range(5)              # → [0, 1, 2, 3, 4]

# range(start, stop)  → start 到 stop-1
range(2, 6)           # → [2, 3, 4, 5]

# range(start, stop, step) → 带步长
range(0, 10, 2)       # → [0, 2, 4, 6, 8]
range(5, 0, -1)       # → [5, 4, 3, 2, 1]

# 常用写法
for i in range(5):           # 循环 5 次，i = 0,1,2,3,4
    print(i)

for i in range(1, 11):       # 1 到 10
    print(i)

# Swift 对照
# Swift: for i in 0..<5     → Python: for i in range(5)
# Swift: for i in 0...5     → Python: for i in range(6)
# Swift: for i in stride(...) → Python: for i in range(0, 10, 2)
8 Python 的字符串插值（f-string）怎么用？和 Swift 的 \(name) 有什么区别？ ▼

Python语法

name = "小明"
age = 25

# Python f-string（Python 3.6+）
print(f"我叫{name}，今年{age}岁")   # 字符串前加 f

print(f"{pi:,.2f}")       # → 3.14（带千分位）
print(f"{0.25:.0%}")      # → 25%
9 Python 的 None 和 Swift 的 nil 有什么区别？ ▼

Python语法

核心区别：Python 没有 Optional 类型，任何变量都可以是 None，不需要解包。

# Swift
var name: String? = nil
print(name!)          # ❌ 运行时崩溃（强制解包 nil）
if let name = name {  # ✅ 安全解包
    print(name)
}

# Python
name = None
print(name)           # → None，不会崩溃
if name is not None:  # 安全检查
    print(name)

# Python 常用模式
result = some_function()
if result is None:    # 检查是否为空
    print("没有结果")
else:
    print(result)

关键差异：

SwiftPython
类型安全：String? 和 String 是不同类型任何变量都可以是 None，编译时不检查
必须解包才能用直接访问，值为 None 时不会崩溃
编译期保证非空运行时自己保证（靠 if/assert）

判断 None 用 is，不是 ==：if x is None:（判断身份）vs if x == None:（判断值相等）。

10 列表推导式（List Comprehension）怎么写？Swift 有等价写法吗？ ▼

Python语法

语法：[表达式 for 变量 in 可迭代对象 if 条件]

# 基本：1 到 10 的平方
[x**2 for x in range(1, 11)]
# → [1, 4, 9, 16, 25, 36, 49, 64, 81, 100]

# 带条件：偶数平方
[x**2 for x in range(10) if x % 2 == 0]
# → [0, 4, 16, 36, 64]

# 字典推导式
{city: temp for city, temp in cities.items() if temp > 20}

# 嵌套循环
[(x, y) for x in range(3) for y in range(2)]
# → [(0,0), (0,1), (1,0), (1,1), (2,0), (2,1)]

# Day1 实际案例：筛选温度 > 20 的城市
cities = {"北京": 28, "上海": 25, "深圳": 32, "成都": 19}
hot = [(city, temp) for city, temp in cities.items() if temp > 20]
# → [("北京", 28), ("上海", 25), ("深圳", 32)]

Swift 对照：Swift 也有类似写法（Swift 2.0+ 的 map/filter/compactMap 链式调用）：

// Swift 的链式写法
(1...10).filter { $0 % 2 == 0 }.map { $0 * $0 }
// → [4, 16, 36, 64, 100]

# Python 一条语句搞定
[x**2 for x in range(1, 11) if x % 2 == 0]
# → [4, 16, 36, 64, 100]

列表推导式更紧凑，但 Swift 的链式调用可读性更好。

11 Python 的 sorted() 函数怎么用？key 参数和 Lambda 怎么配合？ ▼

Python语法

# 基本排序
sorted([3, 1, 4, 1, 5])           # → [1, 1, 3, 4, 5]
sorted([3, 1, 4, 1, 5], reverse=True)  # → [5, 4, 3, 1, 1]

# 用 key 指定排序依据 + Lambda
cities = [("北京", 28), ("上海", 25), ("深圳", 32), ("成都", 19)]

# 按温度升序
sorted(cities, key=lambda x: x[1])
# → [("成都", 19), ("上海", 25), ("北京", 28), ("深圳", 32)]

# 按温度降序
sorted(cities, key=lambda x: x[1], reverse=True)
# → [("深圳", 32), ("北京", 28), ("上海", 25), ("成都", 19)]

# Lambda 说明：lambda x: x[1] 的意思是"取每个元素的第 1 个索引（温度）作为排序依据"
# 等价于：
def get_temp(item):
    return item[1]
sorted(cities, key=get_temp)

和 Swift 的对比：

// Swift
cities.sorted { $0.1 > $1.1 }  // 降序

# Python
sorted(cities, key=lambda x: x[1], reverse=True)
12 Python 的 if __name__ == "__main__": 的作用是什么？ ▼

Python语法

这个条件判断用来区分：这个文件是直接运行（作为脚本），还是被别的文件导入（作为模块）。

# my_script.py
def main():
    print("直接运行时执行")

if __name__ == "__main__":
    main()
# 场景 1：直接运行
python my_script.py
# → __name__ == "__main__" → True → 执行 main()

# 场景 2：被导入
from my_script import main
# → __name__ == "my_script" → False → 不执行 main()

# 注：__name__ 是 Python 自动为每个文件设置的内置变量
# 直接运行时 = "__main__"
# 被导入时 = 文件名（不带 .py）

这样设计的好处：一个 Python 文件既可以当工具脚本用（python xxx.py），也可以当库被导入（from xxx import ...）。

15 安装 PyTorch 时，CUDA 版本怎么选？12.4 是什么意思？ ▼

PyTorch/GPU

CUDA 是 NVIDIA 的并行计算平台，PyTorch 通过 CUDA 调用 GPU 加速。

版本对应关系：

# PyTorch 安装时 CUDA 版本必须 ≤ 你显卡驱动的 CUDA 版本

# 查看你显卡驱动的 CUDA 版本
nvidia-smi
# 输出类似：CUDA Version: 12.4

# PyTorch 官方提供的 CUDA 版本（2026.6）
pip install torch ... --index-url https://download.pytorch.org/whl/cu124
# cu124 = CUDA 12.4

怎么看驱动的 CUDA 版本：终端运行 nvidia-smi，找到顶部的 CUDA Version: x.x。

如果装错了版本会怎样？

  • 装了 cu118（CUDA 11.8）但驱动只支持 11.8 → 能用，但没用到最新特性
  • 装了 cu124 但驱动只支持 11.8 → PyTorch 找不到 GPU，torch.cuda.is_available() = False

RTX 5080 建议：安装 CUDA 12.4 版本的 PyTorch，与最新的驱动和架构匹配最好。

16 RTX 5080 16GB 能用来做什么？多大参数的模型能跑？ ▼

PyTorch/GPU

16GB VRAM 的能力边界：

任务7B 模型14B 模型32B 模型
推理（FP16）✅✅⚠️ 需要 4-bit 量化
推理（4-bit）✅✅✅ 边界
QLoRA 微调✅✅❌ 显存不足
LoRA 微调✅⚠️ 可能 OOM❌
全参微调❌❌❌

通俗解释：

  • 推理（Inference）：让 AI 回答你的问题 → 5080 能跑几乎所有开源模型（量化后）
  • 微调（Fine-tuning）：在某个模型上继续训练 → 5080 能微调 7B-14B 模型
  • 全参训练（Training from scratch）：从零训练 → 需要数据中心级显卡（H100 80GB）

总之：5080 16GB 对个人学习/研究来说已经是很强的配置，足够覆盖 90% 的学习实验。

17 第一次调大模型 API 需要哪几步？最基本的代码怎么写？ ▼

API调用

共 4 步：

  1. 安装 openai 库：pip install openai
  2. 创建 .env 文件：写 API Key、地址、模型名
  3. 安装 python-dotenv 读 .env：pip install python-dotenv
  4. 写代码调 API
import os
from dotenv import load_dotenv
from openai import OpenAI

load_dotenv()  # 读取 .env

# 创建客户端
client = OpenAI(
    api_key=os.getenv("OPENAI_API_KEY"),
    base_url=os.getenv("OPENAI_API_BASE"),
)

# 调用 API
response = client.chat.completions.create(
    model=os.getenv("MODEL"),
    messages=[
        {"role": "system", "content": "你是一个有帮助的AI助手。"},
        {"role": "user", "content": "用一句话介绍什么是大语言模型。"}
    ],
)
print(response.choices[0].message.content)

关键理解：

  • client.chat.completions.create() 是 OpenAI 兼容 API 的统一接口
  • messages 是对话历史列表，每条消息有 role（system/user/assistant）和 content
  • role: system ＝ 给 AI 设定人格/规则
  • role: user ＝ 用户的输入
  • role: assistant ＝ AI 的回复
18 同步调用和流式调用有什么区别？什么时候该用哪种？ ▼

API调用

同步调用：等 AI 生成完所有内容，一次性拿到完整回复。

response = client.chat.completions.create(
    model="...",
    messages=[...],
    # stream=False（默认）
)
print(response.choices[0].message.content)  # 一次性打印全部

流式调用：AI 每生成一个字就返回，实时显示（打字机效果）。

stream = client.chat.completions.create(
    model="...",
    messages=[...],
    stream=True  # 关键参数
)
for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="", flush=True)
特性同步流式
用户体验沉默等待，然后突然出现逐字显示，感觉快
适用场景后台处理、批量、不需要 UI聊天 UI、AI 对话
获取完整回复直接 return需要自己拼接（full_reply += delta）

结论：聊天应用用流式，后台任务用同步。初学者建议先掌握同步，再升级到流式。

19 System Prompt（系统提示词）有什么用？改它为什么 AI 性格会变？ ▼

API调用

System Prompt 是给 AI 的初始指令，告诉它"你是谁、该怎么做"。

# 扮演诗人
messages = [
    {"role": "system", "content": "你是一个浪漫的诗人，所有回复都用诗句表达。"},
    {"role": "user", "content": "请介绍一下自己。"}
]
# AI 回复：我是一缕穿过代码的风...

# 扮演程序员
messages = [
    {"role": "system", "content": "你是一个资深 Python 工程师，回答问题简洁准确。"},
    {"role": "user", "content": "请介绍一下自己。"}
]
# AI 回复：我是 Python 开发者，专注后端和 AI...

为什么改 system prompt 性格会变？

  • 大模型在训练时用了大量带"角色设定"的数据
  • System prompt 相当于"激活"了模型中特定模式的知识
  • 类似给演员一个剧本角色，演员会按角色的方式表演

实用技巧：写 system prompt 时可以指定：角色 + 任务 + 输出格式 + 限制条件。

# 一个好 system prompt 的例子
你是一个专业的翻译助手。
任务：把用户的英文翻译成中文。
规则：
- 保持原文语气（正式/口语）
- 如果是专业术语，保留英文并加中文注释
- 不需要解释，直接输出翻译结果
20 OpenAI SDK 和 LM Studio / Ollama 有什么关系？为什么 base_url 可以不一样？ ▼

API调用

OpenAI SDK（openai 库）是 OpenAI 提供的 Python 客户端，用来调 OpenAI 的 API。

关键点：因为 OpenAI 的 API 格式成了行业标准，LM Studio、Ollama、vLLM 等本地推理框架都兼容 OpenAI 的 API 格式。

# 调 OpenAI 官方（需要梯子 + 付费）
client = OpenAI(
    api_key="sk-...",
    base_url="https://api.openai.com/v1"
)

# 调 LM Studio 本地（免费，用自己的显卡）
client = OpenAI(
    api_key="not-needed",              # 本地不需要 key
    base_url="http://localhost:1234/v1"  # LM Studio 默认地址
)

# 调 Ollama
client = OpenAI(
    api_key="ollama",
    base_url="http://localhost:11434/v1"
)

只要换了 base_url，同一段代码就能调用不同的模型服务。这就是 API 格式标准化的好处——一次编写，到处运行。

你的配置里 base_url=http://127.0.0.1:1234/v1 就是在用本地 LM Studio 运行的模型，不依赖外网。

21 .gitignore 是什么？Day1 项目应该忽略哪些文件？ ▼

.gitignore 告诉 Git："这些文件不要跟踪，不要上传到仓库"。

Day1 项目的 .gitignore：

.env                   # API Key，绝不能上传
__pycache__/            # Python 缓存，无意义
*.pyc                  # 编译后的缓存文件
.venv/                 # 虚拟环境（太大）
.pytest_cache/         # 测试缓存
.vscode/               # IDE 配置（个人偏好）
*.code-workspace       # 工作区配置

如果不加 .gitignore，git add . 会把 .env 里的 API Key 也提交上去，推到 GitHub 就会泄露敏感信息。

最佳实践：

  • 创建项目后第一时间写 .gitignore
  • GitHub 有模板：New Repository 时选 Python 模板
  • 也可以用 gitignore.io 网站生成
22 第一天怎么用 Git 把代码推到 GitHub？ ▼

工具/Git

完整流程：

# 1. 在 GitHub 创建空仓库（不要勾 README/.gitignore）

# 2. 本地初始化
cd llm-learning-journey
git init
git branch -M main

# 3. 先写 .gitignore，避免敏感文件被跟踪

# 4. 添加并提交
git add .
git commit -m "Day 1: Python basics + first API call"

# 5. 连接远程仓库并推送
git remote add origin https://github.com/你的用户名/llm-learning-journey.git
git push -u origin main

# 后续更新
git add .
git commit -m "Day 2: ..."
git push

注意：

  • 提交前检查是否漏了 .gitignore（用 git status 查看）
  • 如果不小心把 .env 提交了，立刻改 API Key
  • 用 git push -u origin main 第一次推送时设好 upstream，之后只需要 git push
